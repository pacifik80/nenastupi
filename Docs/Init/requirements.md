# Техническое задание: Сервис проверки надёжности работодателя

**Версия:** 1.0  
**Дата:** 29 января 2026 г.  
**Статус:** Утверждено для проектирования  

---

## 1. Бизнес-цель

Создать сервис, позволяющий соискателям оценивать риски трудоустройства в компании на основе объективных данных из открытых источников, включая:
- юридическую чистоту (банкротства, судебные дела)
- финансовую стабильность (косвенные показатели)
- репутационные риски (новости, отзывы)
- структуру связей (дочерние компании, ключевые лица)

**Ключевая метрика успеха:** Снижение количества случаев трудоустройства соискателей в компании с признаками мошенничества или нестабильности на 30% (измеряется через опросы пользователей через 6 месяцев после запуска).

---

## 2. Целевая аудитория

| Сегмент | Потребность | Частота использования |
|---------|-------------|----------------------|
| Соискатели (22–45 лет) | Быстрая проверка перед собеседованием | 1–3 раза в месяц |
| Студенты-выпускники | Проверка первого работодателя | 1–2 раза в год |
| Фрилансеры | Верификация заказчика перед началом работы | 5–10 раз в месяц |

---

## 3. Функциональные требования

### 3.1. Основные сценарии использования

| ID | Сценарий | Приоритет | Критерий приёмки |
|----|----------|-----------|------------------|
| FR-01 | Пользователь вводит название компании или ИНН/ОГРН | P0 | Система возвращает результат за ≤5 секунд |
| FR-02 | Система определяет юридический статус компании (действующая/ликвидированная/банкрот) | P0 | Статус актуален на дату запроса (данные обновлены за последние 24 ч) |
| FR-03 | Система выявляет признаки финансовой нестабильности | P0 | Обнаруживает ≥3 из: банкротство, долги ФССП, массовые увольнения в новостях, задержки ЗП в отзывах |
| FR-04 | Система строит связи компании с другими юрлицами и лицами | P1 | Отображает дочерние компании, материнские структуры, ключевых руководителей за последние 5 лет |
| FR-05 | Система генерирует текстовый отчёт с рекомендациями | P0 | Отчёт содержит 3–5 ключевых рисков и 2–3 действия для верификации перед подписанием договора |
| FR-06 | Пользователь получает уведомление о новых рисках для ранее проверенной компании | P2 | Push-уведомление в Telegram при появлении банкротства или массовых сокращений |

### 3.2. Источники данных и их характеристики

| Источник | Тип данных | Доступность | Частота обновления | Ограничения |
|----------|------------|-------------|-------------------|-------------|
| **ФНС (ЕГРЮЛ/ЕГРИП)** | ИНН, ОГРН, статус, адрес, руководитель | Открытый API (egrul.nalog.ru) | Ежедневно | Лимит 100 запросов/час без авторизации |
| **ЕФРСБ** | Банкротства, процедуры | Открытый реестр (fedresurs.ru) | В реальном времени | Парсинг HTML (без официального API) |
| **ФССП** | Исполнительные производства | Открытый реестр (fssp.gov.ru) | Еженедельно | Требует ФИО + дата рождения для физлиц |
| **Картотека арбитражных дел** | Судебные дела | Открытый реестр (kad.arbitr.ru) | В реальном времени | Поиск только по ИНН/ОГРН |
| **Госзакупки (ФЗ-44/223)** | Контракты, заказчики/поставщики | Открытый API (zakupki.gov.ru) | Ежедневно | Только для компаний-участников госзакупок |
| **Яндекс.Новости** | Новостной поток | API (яндекс.новости) | В реальном времени | Лимит 1000 запросов/день (бесплатный тариф) |
| **hh.ru** | Вакансии, динамика найма | API (api.hh.ru) | Ежедневно | Лимит 500 запросов/сутки, запрет парсинга |
| **Telegram-каналы (публичные)** | Инсайды, утечки | Парсинг через Telegram API | В реальном времени | Только публичные каналы, запрет на приватные |

> ⚠️ **Критическое ограничение**: Запрещено использование данных, полученных в обход `robots.txt` или условий использования сервисов (например, парсинг закрытых разделов hh.ru).

## 4. Нефункциональные требования

| Категория | Требование | Метрика |
|-----------|------------|---------|
| **Производительность** | Время ответа на запрос проверки компании | ≤5 секунд (p95) |
| **Доступность** | Готовность сервиса | ≥99.5% в течение месяца |
| **Масштабируемость** | Поддержка пиковой нагрузки | 100 одновременных запросов |
| **Безопасность** | Защита персональных данных | Соответствие ФЗ-152 (РФ), шифрование данных в покое и при передаче |
| **Юридическая чистота** | Отсутствие нарушений авторских прав | Все данные только из открытых источников или через официальные API |
| **Локализация** | Поддержка русского языка | 100% интерфейса и отчётов на русском |

---

## 5. Юридические ограничения (РФ)

1. **Хостинг**: Серверная инфраструктура должна находиться на территории РФ (требование ФЗ-242 для обработки ПДн граждан РФ).
2. **Регистрация**: Сайт подлежит уведомлению в Роскомнадзоре как информационный ресурс.
3. **Персональные данные**: Запрещено хранение ФИО пользователей без согласия. Допустимо хранение:
   - Хэшей названий компаний (для кэширования)
   - Анонимизированных логов запросов (без привязки к пользователю)
4. **Интеллектуальная собственность**: Запрещено копирование текстов новостей. Допустимо хранение:
   - Ссылок на источники
   - Кратких выдержек (до 10% объёма) с указанием авторства

## 6. Критерии качества данных

| Атрибут | Требование | Метод верификации |
|---------|------------|-------------------|
| **Актуальность** | Данные о статусе компании не старше 24 часов | Сравнение с датой последнего обновления в источнике |
| **Полнота** | Для ≥95% российских компаний найден ИНН/ОГРН | Выборка из 1000 компаний из списка Росстата |
| **Точность связей** | Ошибки в определении дочерних компаний ≤5% | Верификация 100 случайных связей вручную |
| **Релевантность новостей** | ≥80% новостей в отчёте относятся к финансовой/юридической деятельности | Экспертная оценка 50 отчётов |

---

## 7. Матрица рисков работодателя (онтология угроз)

Система должна распознавать и ранжировать следующие категории рисков:

| Категория | Индикаторы | Источник данных | Вес риска |
|-----------|------------|-----------------|-----------|
| **Юридический** | Банкротство, ликвидация, массовые иски от сотрудников | ЕФРСБ, картотека | Критический |
| **Финансовый** | Долги ФССП > 10 млн ₽, задержки ЗП в отзывах | ФССП, отзывы | Высокий |
| **Репутационный** | Негатив в СМИ о мошенничестве, обмане клиентов | Яндекс.Новости | Средний |
| **Операционный** | Массовые сокращения (>15% штата за квартал) | Новости + динамика вакансий | Средний |
| **Структурный** | Связь с банкротыми компаниями через руководителей | Граф связей | Умеренный |

> ⚠️ **Требование к алгоритму**: Вес риска должен корректироваться на основе:
> - Временной близости события (событие 3 месяца назад = 0.7 веса от текущего)
> - Достоверности источника (официальный реестр = 1.0, анонимный отзыв = 0.3)

---

## 8. Ограничения и допущения

| Тип | Описание |
|-----|----------|
| **Географическое** | Первичная поддержка только для компаний РФ (ИНН/ОГРН). Глобальные компании — только при наличии российского юрлица. |
| **Тип компаний** | Фокус на юридических лицах (ООО, АО). ИП поддерживаются, но с ограниченным набором индикаторов. |
| **Данные о зарплатах** | Не используются напрямую (недостоверны). Косвенные индикаторы: динамика вакансий, отзывы о задержках. |
| **Связи «человек → компания»** | Ограничены 5-летним периодом (для снижения шума в графе). |
| **Язык интерфейса** | Только русский (английский не требуется на этапе MVP). |

---

## 9. Критерии приёмки MVP

Сервис считается готовым к запуску, если:

- [ ] Пользователь получает отчёт по компании по ИНН/ОГРН за ≤5 секунд
- [ ] Отчёт содержит статус компании из ЕГРЮЛ (актуальный на дату запроса)
- [ ] Отчёт содержит информацию о банкротствах из ЕФРСБ (если есть)
- [ ] Отчёт содержит ≥3 релевантных новостных упоминания за последние 90 дней
- [ ] Отчёт содержит 3–5 ключевых рисков с указанием источника данных
- [ ] Сервис доступен как веб-сайт и как Telegram-бот
- [ ] Все данные получены легально (через открытые реестры или официальные API)
- [ ] Нет хранения ПДн пользователей без согласия

## Приложение A: Онтология сущностей

Система должна оперировать следующими сущностями и их атрибутами:

| Сущность | Обязательные атрибуты | Источник идентификатора |
|----------|----------------------|-------------------------|
| **Компания** | ОГРН (первичный), ИНН, полное название, статус, дата регистрации | ФНС (ЕГРЮЛ) |
| **Человек** | ФИО, ИНН (для ИП), дата рождения (опционально) | ФНС, СПАРК |
| **Связь (компания → компания)** | Тип (дочерняя/материнская/подрядчик), доля владения (опционально), период действия | Госзакупки, СПАРК |
| **Связь (человек → компания)** | Роль (руководитель/учредитель), период работы, доля владения | ФНС, СПАРК |
| **Событие** | Тип (банкротство/судебное дело/новость), дата, источник, краткое описание | ЕФРСБ, картотека, Яндекс.Новости |

> ⚠️ **Требование**: ОГРН является первичным идентификатором для юрлиц РФ. ИНН используется только как дополнительный атрибут или для сопоставления с источниками без ОГРН.
